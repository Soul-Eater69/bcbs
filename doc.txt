As an IDP user,

I want the Theme Generator DS/API to generate Theme draft content from an IDMT ticket,

So that I can quickly review and manually enter the recommended Theme details in Jira.

The API should generate:
- Theme summary
- Theme description
- Business Needs
- Business Value Stream
- Value Stream Stage
- L2 Capability
- L3 Capability, when applicable
- confidence/reasoning for each recommendation

The generated values should come only from approved catalogs where applicable. If the API is not confident, it should leave the field blank instead of guessing.

Out of scope:
- Creating Jira tickets automatically
- Updating Jira directly
- UI changes


Acceptance Criteria:

1. The system should generate draft Theme content from the IDMT ticket.

2. The generated content should include:
   - Theme description
   - Business Needs
   - Title
   - Value Stream Stage
   - L2 Capability
   - L3 Capability, when applicable

3. The generated Theme description, Business Needs, and other fields should follow the formatting and structure used in previous Jira ticket templates.

4. The generated content should be provided as draft recommendations for user review and manual entry into Jira.
