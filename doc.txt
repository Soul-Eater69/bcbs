Subject: Stage extraction — Theme→Epic link issue and fix

Hi Seetha,

Quick update on the stage (Epic) extraction from Themes.

Issue: In Jira, a Theme (GROUP issue) is connected to its Epics through the parent–child relationship, not through issue links. So when we read a Theme's inward/outward issue links, the child Epics don't appear there — the extraction was coming back with no stages for those Themes.

Resolution: We switched to a reverse lookup — instead of reading the Theme's links, we query for the Epics that point up to the Theme as their parent:

parent = "<THEME-KEY>" AND issuetype = Epic

This returns all child Epics directly. To be robust across project configurations we also union the "Parent Link" custom field, dedupe by Epic key, and skip Cancelled Epics. Each Epic's Value Stream Stage field then gives us the stage.

This is working as expected now and the Themes are returning their stages.
