Success Criteria (Vocabulary Entity Match + Filter Retrieval)

The system performs:

• Extract entity names from a controlled vocabulary file
• Match entities against the LLM response text using RapidFuzz
• Select the top matched entities above the cutoff score
• Build a single OData filter expression using matched entities
• Call the search API once with the filter to retrieve relevant documents

Edge Cases

• If no entities match the response text → return empty result ([])
• If entity length < 4 characters → ignored during matching
• If entity score < cutoff (default 85) → not included
• If more than max_terms matches occur → only top matches are kept
• If search API returns documents with no content → stored as empty string
• Apostrophes in entity names are escaped for OData filters

Output Response Includes

• matched_entities
• matched_entity_scores
• filter_expression used for search
• retrieved_documents (List[Document])

Each Document contains:

• page_content
• metadata
• matched_entities (debug info)
• matched_entity_scores (debug info)
